💡 正確的做法是什麼？

  1. 使用 Git Fork：如需複製修改開源專案，請使用 Git 的 Fork 功能，保留所有歷史紀錄。
  2. 保留原始授權：不要刪除或修改原始 LICENSE 檔案，必須保留完整內容。
  3. 適當標註：在修改的檔案中明確標示變更內容、日期和作者。
  4. 選擇合適授權：如需使用更嚴格的授權條款，請選擇現有的標準授權（如 GPL）而非自創條款。
  5. 遵守 OSI 定義：如果宣稱是「開源」，則應符合開源定義的所有條件。
  6. 法律諮詢：如有特殊需求，請諮詢專業法律人士而非自行修改授權條款。
